# Phase 2 — Step 2.3: Flutter Home/Radar screen

## Objective

Build the consumer Home/Radar screen per DESIGN §7.1 and §8: an animated Stress Score gauge driven by the
live score endpoint, greeting/avatar, live-multibank badge, a pulsing liquidity-deficit warning card,
income-consistency and spending-velocity stat cards, a linked-accounts list from the accounts endpoint,
and the bottom nav (Home · Simulate · Rescue · Goals) — RTL-correct in Arabic.

## Dependencies

- **Requires:** Step 2.2 — `GET /api/v1/clients/{id}/stress-score` returning the enveloped DTO
  (score, zone, color, sub-scores, asOfDate).
- **Requires:** Step 0.3 — Flutter skeleton with the single theme/token file (DESIGN §8 palette, fonts,
  radii, shadows, animations), i18n/RTL setup, Riverpod, and the `dio` API client + envelope unwrapping.
- **Assumes:** the accounts endpoint `GET /api/v1/clients/{id}/accounts` (Phase 1) is available for the
  linked-accounts list.

## Tasks

Complete in order:

1. **API wiring (Riverpod + dio)**
    - Add a data/repository layer + providers that call `GET /api/v1/clients/{id}/stress-score` and
      `GET /api/v1/clients/{id}/accounts`, unwrapping the `{status, data}` envelope into models.
    - Handle loading and error states so the screen degrades gracefully (no crash) when a call fails.

2. **Animated Stress Score gauge**
    - Build an arc gauge with `CustomPainter` (or SVG) showing the three colour zones from DESIGN §8/§5.1
      (`OPTIMAL #1D9E63`, `WARNING #E5A63A`, `CRITICAL #E0574F`) across the 0–100 range.
    - Animate the marker/needle to the live score over **1000 ms** with a **cubic ease-out** curve
      (DESIGN §8 "score animation 1000ms cubic ease-out"). Use the `color` from the DTO for the score
      readout so it matches the served zone.

3. **Header: greeting + avatar + live-multibank badge**
    - Greeting text (localized ar/en) with the client avatar, and a "live multibank" badge indicating
      multiple linked banks (DESIGN §7.1).

4. **Pulsing liquidity-deficit warning card**
    - A warning card using the `bsr-pulse` animation (2.4 s) and the red-warning gradient (DESIGN §8).
    - Render it only when a deficit signal is present; when the forecast data is absent (arrives in
      Phase 3), render gracefully (hidden or neutral placeholder) — see Task 7.

5. **Stat cards: income consistency & spending velocity**
    - Two stat cards bound to `incomeConsistency` and `spendingVelocity` from the stress-score DTO,
      styled per §8 (cards radius 20–24, soft shadow), localized labels.

6. **Linked-accounts list**
    - Render the accounts from the accounts endpoint (bank name, display colour, latest balance) using the
      `money(n)` formatting helper convention from DESIGN §8 (SAR / ر.س).

7. **30-day forecast chart placeholder**
    - Reserve the forecast-chart slot from DESIGN §7.1 but render it gracefully when no data is present
      (empty/skeleton state), since the forecast endpoint arrives in **Phase 3 Step 3.2**. Structure the
      widget so wiring it to the forecast endpoint later is a drop-in.

8. **Bottom navigation**
    - Bottom nav with Home · Simulate · Rescue · Goals (DESIGN §7). Home is active; the other tabs
      navigate to their (possibly placeholder) routes.

9. **RTL / i18n**
    - Verify layout mirrors correctly under Arabic (`Directionality` RTL), using IBM Plex Sans Arabic and
      the theme tokens from Step 0.3 — no hardcoded colours or text directions.

## Acceptance Criteria

All the following must be true before marking this step complete:

- [ ] Home renders with a live score and live linked accounts fetched from the API (no hardcoded values).
- [ ] The gauge shows the three colour zones and animates the marker to the score over ~1000 ms with a
      cubic ease-out curve.
- [ ] The liquidity-deficit warning card visibly pulses (`bsr-pulse`) when shown.
- [ ] Income-consistency and spending-velocity stat cards display the DTO sub-scores.
- [ ] The forecast-chart slot renders gracefully with no data (no error/overflow) pending Phase 3 Step 3.2.
- [ ] Bottom nav (Home/Simulate/Rescue/Goals) is present and navigates between tabs.
- [ ] The screen is RTL-correct in Arabic and uses only the shared theme tokens (DESIGN §8).
- [ ] Loading and API-error states are handled without crashing.

## Notes

- **Forecast dependency:** the 30-day forecast chart's real data source is Phase 3 Step 3.2's forecast
  endpoint; this step only reserves and gracefully renders the slot. Do not block Home on forecast data.
- Keep all colours, fonts, radii, shadows, and animation durations in the single theme/token file from
  Step 0.3 (Global Rules: Flutter design tokens live in one theme file, consumed by both shells).
- The gauge's zone colours and the score readout colour must match the `color` returned by the Step 2.2
  endpoint so the UI and API never disagree.
- Use the `fmt`/`money` formatting helpers from DESIGN §8 for all currency/number display (SAR vs ر.س by
  locale).
