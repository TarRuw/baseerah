# Phase 3 — Step 3.4: GenAiClient interface + MockGenAi + chat/invoice endpoints

## Objective

Provide pluggable conversational AI (FR-03) behind a `GenAiClient` interface with a deterministic
`MockGenAi` default that grounds replies in the client's telemetry and requires no API key, plus the
chat and invoice-parse endpoints. The provider is selected by `GENAI_PROVIDER` so the real
`RemoteGenAi` adapter (Phase 7) can drop in without changing callers (DESIGN §2, §5, §6, §9).

## Dependencies

- **Requires:** Step 1.4 — client/transaction data access must exist so mock replies can be grounded in
  the client's telemetry.
- **Assumes:** Phase 0 API envelope + error handling; Spring bean/config wiring is available.

## Tasks

Complete in order:

1. **`GenAiClient` interface**
    - Create `backend/src/main/java/com/baseerah/genai/GenAiClient.java`.
    - Methods: `ChatReply chat(ChatContext context, String message)` and
      `InvoiceParseResult parseInvoice(byte[] image)` (accept the uploaded bytes / a small wrapper).
    - Define package-local domain types (`ChatContext` carrying client telemetry summary; `ChatReply`
      carrying the reply text; `InvoiceParseResult` carrying the parsed-action stub fields).
    - Controllers must depend only on this interface, never a concrete impl (Global Rule).

2. **`MockGenAi` default implementation (deterministic, telemetry-grounded)**
    - Create `backend/src/main/java/com/baseerah/genai/MockGenAi.java`.
    - Keyword-matched, **deterministic** replies grounded in the client's telemetry:
      - car / lease / vehicle / `2100` and their Arabic equivalents (سيارة / تأجير / مركبة) →
        the scenario answer (affordability/impact framing using the client's numbers).
      - else → a generic **24-month analysis** reply referencing the client's telemetry.
    - Replies must be reproducible (same input → same output; no randomness), so demos work offline
      (DESIGN §9 reliability).
    - `parseInvoice` returns a **stub** parsed-action result (e.g. merchant/amount/category placeholder +
      a suggested action) — no real OCR in this step.

3. **`GENAI_PROVIDER` config + bean selection**
    - Add config (`application.yml` / `@ConfigurationProperties` or `@ConditionalOnProperty`) so
      `GENAI_PROVIDER` = `mock` (default) | `remote` selects the `GenAiClient` bean.
    - `mock` must resolve with **no API key set**. `remote` is reserved for the Phase 7 adapter — wire the
      switch now (it may resolve to a not-yet-implemented placeholder), but do not implement the remote
      call here.
    - Create `backend/src/main/java/com/baseerah/genai/GenAiConfig.java` (bean wiring).

4. **Chat + invoice endpoints**
    - Create `backend/src/main/java/com/baseerah/genai/ChatController.java`.
    - `POST /api/v1/clients/{id}/chat` — body `{ message }` → enveloped `{ reply }` (DESIGN §6).
    - `POST /api/v1/clients/{id}/chat/invoice` — **multipart** image upload → enveloped parsed-action stub.
    - Build the `ChatContext` from the client's telemetry (via Step 1.4 data access) before calling the
      injected `GenAiClient`. Thin controllers.

5. **Tests**
    - Create `backend/src/test/java/com/baseerah/genai/MockGenAiTest.java`.
    - Assert car/lease/vehicle/`2100` + an Arabic equivalent route to the scenario reply; other input
      routes to the 24-month generic reply; identical input yields identical output (deterministic).
    - Assert the mock bean resolves with no API key configured.

## Acceptance Criteria

All the following must be true before marking this step complete:

- [ ] `POST /api/v1/clients/{id}/chat` returns a deterministic, telemetry-grounded mock reply with **no
      API key set**.
- [ ] Keyword matching works for car/lease/vehicle/`2100` and their Arabic equivalents (→ scenario) vs.
      other messages (→ 24-month analysis); identical input yields identical output.
- [ ] `POST /api/v1/clients/{id}/chat/invoice` accepts a multipart image and returns a parsed-action stub
      in the standard envelope.
- [ ] `GENAI_PROVIDER=mock` (default) resolves the `MockGenAi` bean; `GENAI_PROVIDER=remote` resolves the
      reserved remote bean/placeholder via the same switch — controllers depend only on `GenAiClient`.
- [ ] `MockGenAiTest` passes and all pre-existing tests still pass.
