# Phase 5 — Step 5.3: Flutter Goals screen

## Objective

Build the consumer **Goals** screen (DESIGN §7.4 and §8): a gold reward-points card with the client's
risk tier and a list of challenge cards with progress bars and a claim button — driven live by the
challenge/rewards endpoints so claiming a completed challenge updates points and button state on screen.

## Dependencies

- **Requires:** Step 5.2 — `GET /api/v1/clients/{id}/challenges`, `POST /api/v1/clients/{id}/challenges/{cid}/claim`,
  and `GET /api/v1/clients/{id}/rewards` must be live and enveloped. **Step 0.3** — the Flutter skeleton
  (theme tokens, i18n/RTL, routing, dio client, bottom nav with the Goals tab) must exist.
- **Assumes:** Riverpod (or the state-mgmt choice fixed in Phase 0), `dio` HTTP client, and the shared
  theme file with the DESIGN §8 tokens (teal/gold palette, gradients, radii, fonts, `bsr-up` animation)
  are in place; envelope unwrapping is handled by the shared client layer.

## Tasks

Complete in order:

1. **API client + models**
    - Add a gamification API client (dio) with methods: `fetchChallenges(clientId)`,
      `claimChallenge(clientId, challengeId)`, `fetchRewards(clientId)`, unwrapping the `{status,data}`
      envelope.
    - Dart models mirroring the DTOs: `Challenge { id, icon, title, subtitle, reward, pct, progressText,
      claimable, claimed }` and `Rewards { points, riskTier }`.

2. **State / providers**
    - A Goals controller/notifier holding the rewards summary + challenge list, with a `claim(challengeId)`
      action that calls the endpoint and updates points + the affected challenge's `claimed`/`claimable`
      state from the response (optimistic or on-response — pick one and keep loading/error states).

3. **Gold reward-points card — DESIGN §7.4 / §8**
    - A card using the gold gradient (`135deg #C4A24C→#a8863a`, token from the theme) showing the total
      **points** prominently and a **risk-tier badge**. Format the number with the §8 `fmt` helper.

4. **Challenge cards — DESIGN §7.4**
    - For each challenge: leading `icon`, `title` + `subtitle`, the `reward` (points), a **progress bar**
      that fills to `pct` and transitions gold → success green (`#1D9E63`) when complete (`pct >= 100`),
      the `progressText`, and an action button whose label/state is:
        - **In-progress** (disabled) when `pct < 100 && !claimed`,
        - **Claim** (enabled) when `claimable` (`pct >= 100 && !claimed`),
        - **Claimed ✓** (disabled) when `claimed`.

5. **Live claim flow**
    - Tapping **Claim** calls the claim endpoint; on success the points card updates live and the button
      flips to **Claimed ✓**. Surface a non-blocking error (and leave state unchanged) if the claim is
      rejected.

6. **RTL / i18n**
    - Screen renders correctly RTL for Arabic (`ar`): progress bars, icons/badges, and the card layout
      mirror correctly; all strings go through the i18n layer (ar/en), currency/number via the §8 helpers.

7. **Widget test**
    - A widget test with a mocked API: the screen renders the points card + challenge list, a completed
      challenge shows an enabled **Claim** button, and tapping it (mock returns updated points + claimed)
      updates the displayed points and flips the button to **Claimed ✓**.

## Acceptance Criteria

All the following must be true before marking this step complete:

- [ ] The Goals screen lists live challenges from `GET .../challenges`, each with a progress bar filled to
      `pct` and the correct button state (In-progress / Claim / Claimed ✓).
- [ ] The gold reward-points card shows the total `points` and the risk-tier badge from `GET .../rewards`.
- [ ] A completed challenge's progress bar renders in success green; an in-progress one renders gold.
- [ ] Tapping **Claim** on a completed challenge calls the claim endpoint and, on success, updates the
      displayed points live and flips the button to **Claimed ✓**; a rejected claim leaves state unchanged.
- [ ] The screen renders correctly in RTL (Arabic) and pulls all copy through i18n; numbers use the §8
      formatting helpers.
- [ ] A widget test covering render + claim flow passes; existing Flutter tests still pass.
