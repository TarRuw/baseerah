# Phase 8 — Step 8.3: Consumer UI polish (pluralization, recovery gauge, stat colour, locale persistence)

## Objective

Fix the batch of low-severity consumer-shell defects from the QA review: **UI-02** (bank-count
pluralization), **UI-03** (rescue recovery gauge shows the wrong score), **UI-04** (spending-velocity colour
at max value), and **UI-07** (language choice not persisted across a full reload). Reference:
[`docs/QA_E2E_FINDINGS.md`](../../docs/QA_E2E_FINDINGS.md) UI-02, UI-03, UI-04, UI-07.

## Dependencies

- **Requires:** Home/Radar (Phase 2.3), Rescue screen (Phase 4.3), and the locale toggle/provider
  (Phase 0.3). `frontend/lib/l10n/app_en.arb` + `app_ar.arb` and `frontend/lib/core/locale_provider.dart`
  exist.

## Tasks

Complete in order:

1. **UI-02 — Fix bank-count pluralization.**
    - The "Live · N banks" badge shows `1 banks` / `1 بنوك`. Use ICU plural in the ARB strings
      (`{count, plural, one{...} other{...}}`) so count 1 reads `1 bank` / `بنك واحد` and N reads correctly.
    - Update the Home badge widget to pass the count through the localized plural message.

2. **UI-03 — Rescue recovery gauge shows the recovered score.**
    - On the Rescue "complete" state, the recovery gauge's large number currently shows the **before** score
      (e.g. 83) while the caption says "recovers 83 → 91". Change the gauge to display/animate to the
      **projected post-rescue** score (91) — or, if the before-score is intentional, relabel it explicitly as
      "current" and show 91 as the target. Prefer showing the recovered score to match the "recovery" framing.
    - Use the `projectedScore`/recovery value already returned by the rescue-confirm response; do not hardcode.

3. **UI-04 — Spending-velocity stat colour at the extreme.**
    - On Home, "Spending velocity = 100" renders an **orange** bar while "Income consistency = 100" is green.
      Review the colour-threshold mapping for spending velocity and align the bar colour with the metric's
      intended good/bad direction so a top value isn't shown as a caution colour. Keep thresholds in the theme.

4. **UI-07 — Persist language selection across reloads.**
    - The chosen locale is currently in-memory only, so a full page reload reverts to Arabic. Persist the
      selected locale (e.g. `shared_preferences`) in `locale_provider.dart` and restore it on app boot.

5. **Tests.**
    - Widget/unit tests: plural renders `1 bank`/`بنك واحد` at count 1; recovery gauge reflects the recovered
      score; locale provider restores a persisted value on init.

## Acceptance Criteria

- [ ] The bank-count badge reads `1 bank` / `بنك واحد` at count 1 and pluralizes correctly for N>1 in both
      languages.
- [ ] After confirming a rescue, the recovery gauge communicates the **recovered** score (matches the "→ Y"
      caption) — no contradiction between the number and the caption.
- [ ] Spending-velocity bar colour matches its intended direction at the extremes (no caution colour for a
      top/healthy value).
- [ ] Switching language then doing a full reload keeps the chosen language.
- [ ] New tests pass; no regression to existing Home/Rescue widget tests.
