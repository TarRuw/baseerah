# Phase 0 — Step 0.3: Flutter skeleton — theme, i18n/RTL, routing, dio client

## Objective

Stand up a bootable Flutter app carrying the Baseerah design system, Arabic-first i18n/RTL, both shell
route trees (consumer + bank) as placeholders, and a dio API client — so every UI phase has a themed,
localized, navigable foundation driven by the backend.

## Dependencies

- **Requires:** Step 0.1 — the `frontend/` directory and root tooling must exist; Riverpod is the recorded
  state solution.
- **Assumes:** Flutter 3.x / Dart 3 SDK installed (README prerequisites). The backend need not be running
  for this step; the API client is wired but not yet exercised.

## Tasks

Complete in order:

1. **Scaffold the Flutter project**
    - Run `flutter create` targeting `frontend/` with org `com.baseerah` (app id `com.baseerah.baseerah`).
    - Keep mobile + web platforms enabled (consumer = mobile, bank portal = web/tablet per DESIGN.md §7).

2. **Add dependencies to `pubspec.yaml`**
    - `flutter_riverpod`, `dio`, `go_router`, `intl`, `google_fonts`.
    - Enable `flutter.generate: true` for gen-l10n; add `flutter_localizations` (SDK).
    - Run `flutter pub get`.

3. **Centralize the design system in one theme file**
    - `frontend/lib/theme/baseerah_theme.dart` — hold ALL tokens from DESIGN.md §8, do not scatter them:
      - Colors: primary teal `#0e6b54`→`#0a3d33`, gold `#C4A24C`, alert red `#E0574F`, success green
        `#1D9E63`, warning orange `#E5A63A`, dark text `#13241F`, muted `#6E7F78`, light bg `#F6F4EE`,
        app dark bg `#0c1512`.
      - Fonts: `IBM Plex Sans` (Latin) + `IBM Plex Sans Arabic` via `google_fonts`; expose a text theme.
      - Radii: phone frame 52/40, cards 20–24, buttons/inputs 9–14, avatar = circle.
      - Shadows: soft `0 2px 6px rgba(0,0,0,.06)`, medium `0 8px 24px -16px rgba(10,61,51,.25)`.
      - Gradients: teal `135° #0e6b54→#0a3d33`, gold `135° #C4A24C→#a8863a`, red-warning `135° #c0433c→#8f2f2a`.
    - Build a `ThemeData` (light) and expose the raw tokens as named constants so both shells consume one source.

4. **Set up i18n + RTL**
    - `frontend/lib/l10n/app_en.arb` and `app_ar.arb` with an initial key set (app title, the 4 consumer +
      3 bank nav labels, a language-toggle label). Arabic is the primary/default locale.
    - `l10n.yaml` config for gen-l10n.
    - `MaterialApp.router` with `localizationsDelegates` + `supportedLocales` (`ar`, `en`), and a
      Riverpod-managed locale provider. Wrap content so direction follows locale (RTL for `ar`, LTR for `en`)
      via `Directionality` / `MaterialApp`'s automatic locale-based direction.

5. **Configure routing for both shells with placeholder screens**
    - `frontend/lib/router/app_router.dart` using `go_router`.
    - Consumer shell (bottom nav: Home · Simulate · Rescue · Goals) — routes `/home`, `/simulate`,
      `/rescue`, `/goals` (DESIGN.md §7).
    - Bank shell (sidebar: Applications · Portfolio · Settings) — routes `/bank/apps`, `/bank/portfolio`,
      `/bank/settings`.
    - Each route renders an empty placeholder screen (title + "coming soon"); real screens land in later phases.
    - Include the Consumer/Bank segmented switch conceptually so the two shells are reachable.

6. **Create the dio API client**
    - `frontend/lib/api/api_client.dart` — a dio wrapper with `baseUrl` `http://localhost:8080/api/v1`
      (configurable), default `Accept-Language` header bound to the current locale (`ar`/`en`), and JSON
      content type. Expose it via a Riverpod provider so screens can read it.
    - Do not implement feature calls yet — just the configured client.

## Acceptance Criteria

All the following must be true before marking this step complete:

- [ ] `flutter pub get` succeeds and `flutter run` builds and launches the app.
- [ ] Toggling the language flips the UI between LTR (en) and RTL (ar), and swaps localized nav labels.
- [ ] All design tokens from DESIGN.md §8 live in `lib/theme/baseerah_theme.dart` (single source); no
      hardcoded colors/fonts in screen files.
- [ ] Both shells' placeholder routes (`/home`, `/simulate`, `/rescue`, `/goals`, `/bank/apps`,
      `/bank/portfolio`, `/bank/settings`) navigate without error.
- [ ] `ApiClient` is constructed with base URL `.../api/v1` and sends an `Accept-Language` header matching
      the active locale, and is exposed through a Riverpod provider.
- [ ] `flutter analyze` reports no errors.

## Notes

- Arabic-first: default locale is `ar`; the language toggle is a visible control in the shared toolbar
  (DESIGN.md §7). RTL correctness is a hard requirement (ORCHESTRATION Global Rules).
- Keep tokens centralized so the two shells stay visually consistent (DESIGN.md §8 closing note).
- The API client's base URL should be overridable (e.g. a const or `--dart-define`) so it can point at a
  deployed backend later; localhost is the dev default matching Step 0.2.
- Placeholder screens are intentionally minimal — Phases 2–6 replace them with the real 7 screens.
