# Session Handoff — Step 3.5: Flutter Simulate screen (loan sliders + AI chat tabs)

**Date:** 2026-07-03
**Status:** completed

## What Was Done

Built the consumer **Simulate** screen (DESIGN §7.2/§8) — the last step of Phase 3 — as a two-tab
screen driven entirely by the Phase-3 backend (`loan-simulate` from Step 3.3, `chat` + `chat/invoice`
from Step 3.4). Follows the Home feature's established layering: `data/` (models + one repository that
unwraps the shared `{status,data}` envelope), `state/` (Riverpod), widgets that only read state.

- **`features/simulate/data/`**
  - `loan_result.dart` — client view of `LoanSimulateResponse`. Parses the server-resolved `dtiColor`/
    `verdictColor` hexes (via `BaseerahTokens.hex`) rather than re-deriving them, so the rendered bands
    always agree with the backend verdict (same rule `StressScore` follows).
  - `chat_message.dart` — a tiny `{role, text}` value (`ChatRole.user|ai`); invoice replies render as an
    AI message too, so one flat type covers the transcript.
  - `invoice_result.dart` — client view of `InvoiceParseResult` (the deterministic mock stub).
  - `simulate_repository.dart` — the three dio calls: `simulate` (JSON body), `chat` (JSON body →
    `data.reply`), `parseInvoice` (**multipart** part `image` via `FormData.fromMap` — dio sets the
    `multipart/form-data` content-type + boundary, overriding the client's JSON default).
- **`features/simulate/state/simulate_providers.dart`**
  - `LoanInputs` (with the prototype's slider ranges/steps + defaults 45000/5.5/36) and `LoanState`
    (`inputs` + `AsyncValue<LoanResult>`).
  - **`LoanController`** — slider setters update `inputs` **instantly** (labels track the thumb) but
    coalesce the API call behind a **350 ms debounce `Timer`**, so a rapid drag makes one request, not
    fifty (DESIGN §9 <2.5 s NFR). Uses `AsyncLoading().copyWithPrevious(...)` so the card keeps its last
    value visible while reloading (no flicker-to-spinner). Reads the repository fresh per fetch so each
    request carries the current locale's `Accept-Language`.
  - **`ChatController`** — append-only transcript + `isTyping`. `send()`/`sendInvoice()` push the user
    turn, show the typing dots, then append the reply. A **500 ms typing floor** keeps the `bsr-blink`
    dots visible even though the mock answers instantly (Step 3.5 note). On backend failure it appends a
    graceful error bubble instead of leaving the dots spinning. Invoice bubble text is formatted by an
    injected `render` callback so l10n/`Fmt` stay in the widget layer, not in state.
  - Reuses Home's `currentClientProvider` (via `currentClientIdProvider`) — one "current persona" across
    the app; a future persona picker drives both screens for free.
- **`features/simulate/widgets/`** — `typing_indicator.dart` (one 1 s `AnimationController` → three
  phase-offset dots, the CSS `bsr-blink` keyframe as a triangle wave `.35↔1`), `chat_bubble.dart`
  (`AlignmentDirectional` + `BorderRadiusDirectional`, so bubbles + tail corners mirror under RTL),
  `loan_affordability_tab.dart` (sliders card + teal gradient result card: instalment, total, DTI bar in
  `dtiColor`, verdict band in `verdictColor`, projected←current score impact), `ask_ai_tab.dart` (header,
  transcript, typing dots, suggestion chips, input row with **image-picker invoice upload** + send;
  auto-scrolls to the latest bubble).
- **`features/simulate/simulate_screen.dart`** — title/subtitle + custom two-pill segmented control
  (matches the prototype's `#EAE7DE` track / teal active pill) + the active tab in an `Expanded`.
- **Wiring** — `router/app_router.dart` Simulate branch now routes to `SimulateScreen` (was a
  placeholder); `pubspec.yaml` adds `image_picker: ^1.1.2`; `l10n/app_en.arb` + `app_ar.arb` gain the
  Simulate strings (incl. `loanTermMonths`/`invoiceParsedBody` ICU placeholders), regenerated via
  `flutter pub get`.
- **Tests** — `test/simulate_screen_test.dart` (3 tests, all green): loan result renders from the faked
  API with the server colours (and exactly one call = debounce coalesced the initial load), the AI tab's
  suggestion chip sends + appends a reply after the typing floor, and the screen lays out RTL under
  Arabic. Backend is stubbed with a `implements SimulateRepository` fake via Riverpod overrides — no
  network. The clock is advanced with explicit `pump(Duration)` because the loading spinner and typing
  dots are infinite animations that never `pumpAndSettle`.

## Decisions Made

- **Debounce lives in `LoanController`, not the widget or repository.** The step explicitly calls for
  debounced slider→API calls. Putting the `Timer` in the notifier keeps API calls out of widgets (Global
  Rule) and off the render path, while the immediate `inputs` update keeps labels smooth. 350 ms is the
  usual slider-debounce sweet spot; it's a single named constant if the demo wants it tuned.
- **Two separate `StateNotifier`s, not one.** The tabs have fundamentally different state shapes —
  reactive (inputs + latest result) vs. append-only (message list + typing). Splitting them keeps each
  small and avoids a combined god-state.
- **`image_picker` for the invoice control.** The step says "image picker"; the prototype has no invoice
  UI, so I added an attach button in the input row. `image_picker` 1.x has endorsed federated plugins for
  mobile/web/desktop, so the local demo runs wherever `flutter run` targets. **iOS caveat:** running on
  iOS would need `NSPhotoLibraryUsageDescription` in `ios/Runner/Info.plist` (not added — out of scope for
  a single-machine demo that isn't targeting the iOS simulator).
- **English AI copy in Arabic RTL (deferred, per Step 3.4 handoff).** The `GenAiClient` interface carries
  no locale, so `MockGenAi` replies in English even under `ar`. The chat *layout* is fully RTL-correct;
  only the reply text is English. If Arabic AI copy is wanted for the demo, thread `Accept-Language`
  through `ChatController`/`ChatService` and give `MockGenAi` an Arabic variant (a Phase-7 / polish item).
- **Verdict glyph mapped from the server hex.** The response carries `verdictColor`, not a band enum, so
  the tab maps green/orange/red → check/warning/block icons to reproduce the prototype's ✅/⚠️/⛔ without
  re-deriving the affordability band on the client.
