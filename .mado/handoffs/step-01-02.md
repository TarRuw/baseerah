## What Was Done

## Decisions Made

- **Feature-flat package layout** (`com.baseerah.client` / `.account` / `.transaction`), not
  `com.baseerah.domain.*`. Matches AGENT.md's convention and the step's file table. **Steps 1.3 and 1.4
  must follow this same layout.**
- **UUID PKs generated app-side** via `@GeneratedValue(strategy = GenerationType.UUID)`. The DB keeps its
  `default gen_random_uuid()` as a fallback for non-JPA inserts, but Hibernate assigns the id before flush
  so object graphs are fully linked in memory (simpler for the 1.3 seeder). `validate` ignores column
  defaults, so there is no conflict.
- **`createdAt` uses Hibernate `@CreationTimestamp`** so the NOT-NULL column is populated app-side on
  insert regardless of the DB default (a plain field would insert NULL and violate the constraint).
- **`category` persists the raw feed string; `Category` is a resolve-time view.** `Category.resolve(raw)`
  is case-insensitive/blank-safe and returns `OTHER` for anything unmodelled — never throws (criterion #4).
  `OTHER` is a sentinel and is **never persisted**, so no data is lost and adding a constant later
  reclassifies old rows for free. New categories are added to the enum, not via a migration.
- **Test verifies against the local compose Postgres, not Testcontainers.** The step only asked for a
  `@SpringBootTest`/`@DataJpaTest` that boots against V1; the project's run model already requires
  `docker-compose up`, so booting against that DB is the simplest faithful check. The test is annotated
  `@EnabledIf(postgresReachable)` so it **skips (not fails)** on a fresh checkout with no DB running.

## Issues Encountered

- **Testcontainers was tried first and abandoned.** Spring Boot 3.3.5 manages Testcontainers 1.19.8, whose
  bundled docker-java 3.3.6 negotiates Docker API v1.32 — rejected with HTTP 400 by the local engine
  (Docker Desktop 4.67 / Engine 29.3.1, min API 1.40). Bumping to Testcontainers 1.20.6 (docker-java 3.4.1)
  fixed the 400, but the runner then could not reach Docker Desktop's non-default socket
  (`~/.docker/desktop/docker.sock`; `/var/run/docker.sock` is root-owned and inaccessible) — neither
  `DOCKER_HOST` nor `~/.testcontainers.properties` took effect in the forked test worker. Rather than
  hard-code a machine-specific socket, the test was switched to the live compose DB (above). If hermetic
  container-per-test is wanted later, revisit once the Docker Desktop socket is exposed to Testcontainers
  (e.g. `DOCKER_HOST` reaching workers, or a rootful `/var/run/docker.sock`).
- Harmless Liquibase WARN at boot: `db/changelog/changes/.gitkeep ... is not a recognized file type`
  (the `includeAll` picks up the placeholder). Not worth removing while the dir still needs the keep file;
  will disappear naturally as more changesets land.
