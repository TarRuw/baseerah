# Handoff — Step 0.3: Flutter skeleton — theme, i18n/RTL, routing, dio client

**Status:** completed
**Date:** 2026-07-02

## What was done

- Scaffolded the Flutter app into `frontend/` via `flutter create --org com.baseerah
  --project-name baseerah --platforms=android,ios,web` (app id `com.baseerah.baseerah`).
- Rewrote `frontend/pubspec.yaml`: added `flutter_riverpod`, `dio`, `go_router`, `intl`,
  `google_fonts`, `flutter_localizations` (SDK), and set `flutter.generate: true` for gen-l10n.
- `lib/theme/baseerah_theme.dart` — single source of every DESIGN.md §8 token: `BaseerahTokens`
  (colors, radii, soft/medium shadows, teal/gold/red-warning/desktop-backdrop gradients) plus
  `BaseerahTheme.light(locale)` composing them into `ThemeData` with the correct font per locale.
- i18n/RTL: `l10n.yaml` + `lib/l10n/app_en.arb` (template) and `app_ar.arb` (Arabic strings);
  `lib/l10n/locale_provider.dart` (Riverpod `StateNotifier<Locale>`, **defaults to `ar`**, `toggle()`
  flips ar↔en). Direction follows locale automatically via `MaterialApp.locale` (no manual `Directionality`).
- `lib/router/app_router.dart` — `go_router` with two `StatefulShellRoute.indexedStack` shells:
  consumer bottom-nav (`/home`, `/simulate`, `/rescue`, `/goals`) and bank sidebar/`NavigationRail`
  (`/bank/apps`, `/bank/portfolio`, `/bank/settings`). Shared toolbar carries the logo, a
  Consumer/Bank `SegmentedButton` switch (hard `context.go` between shells), and the language toggle.
  Every route renders a minimal placeholder (icon + label + "coming soon").
- `lib/api/api_client.dart` — dio wrapper, base URL `http://localhost:8080/api/v1` (overridable via
  `--dart-define=BASEERAH_API_BASE_URL`), `Accept-Language` bound to the active locale; exposed as
  `apiClientProvider` which `ref.watch`es the locale so the header tracks the UI language.
- `lib/main.dart` — `ProviderScope` + `MaterialApp.router`, theme/locale/delegates wired from providers.
- Replaced the default counter `test/widget_test.dart` with a smoke test asserting boot-in-Arabic (RTL)
  and toggle-to-English (LTR) with the localized title swapping.

## Decisions

- **`intl` pinned to `^0.20.2`** (not the `^0.19.0` the step suggested): `flutter_localizations` from the
  SDK hard-pins `intl 0.20.2`, so `pub get` fails otherwise. Functionally equivalent.
- **Fonts sourced purely via `google_fonts`** (`ibmPlexSans` / `ibmPlexSansArabic`) — no bundled font
  assets, so the pubspec `fonts:` section is intentionally omitted (the step listed it as optional).
  Fonts fetch at runtime; acceptable for the local-demo scope.
- **Generated localizations are git-ignored** (`frontend/.gitignore` → `/lib/l10n/app_localizations*.dart`).
  With `generate: true`, `flutter pub get` regenerates them, so a fresh checkout still builds — keeping
  generated code out of source control is the idiomatic choice.
- **`flutter create` added its own `frontend/.gitignore`** — kept as-is (standard Flutter ignores);
  appended only the generated-l10n rule.
- **Shell navigation model:** `StatefulShellRoute.indexedStack` per shell (preserves per-tab state for the
  real screens later); the Consumer/Bank switch is a cross-shell `go`, not a tab.
- **`ColorScheme.fromSeed`** seeded on teal, then overridden with exact palette roles — gives Material 3
  component defaults while keeping DESIGN.md §8 colors authoritative.
