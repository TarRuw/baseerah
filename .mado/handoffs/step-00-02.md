# Handoff — Step 0.2: Spring Boot + Postgres + Liquibase skeleton, docker-compose

**Status:** completed
**Date:** 2026-07-02

## What was done

- Bootstrapped a Gradle (Kotlin DSL) project under `backend/` and committed the **Gradle wrapper**
  (`gradlew`, `gradlew.bat`, `gradle/wrapper/gradle-wrapper.jar` + `.properties`), pinned to Gradle 8.10.2.
- `build.gradle.kts`: Java toolchain 21, Spring Boot `3.3.5` + Spring dependency-management `1.1.6`, group
  `com.baseerah`. Deps: web, data-jpa, validation, actuator, `liquibase-core`, `postgresql` (runtimeOnly),
  `spring-boot-starter-test` (test) + `junit-platform-launcher` (testRuntimeOnly).
- `BaseerahApplication.java` — standard `@SpringBootApplication` entry point.
- `application.yml` — `dev` profile active by default; datasource → `jdbc:postgresql://localhost:5432/baseerah`
  (`baseerah`/`baseerah`); `ddl-auto: validate`; `open-in-view: false`; Liquibase enabled pointing at the
  master changelog; actuator exposes only `health` (`show-details: never`). `dev` profile turns `show-sql` on.
- `backend/docker-compose.yml` — `postgres:16`, db/user/pass all `baseerah`, port `5432:5432`, named volume
  `baseerah-pgdata`, plus a `pg_isready` healthcheck.
- Liquibase master changelog (`db/changelog/db.changelog-master.yaml`) uses `includeAll` over `changes/`
  with `relativeToChangelogFile: true`; `changes/.gitkeep` keeps the (currently empty) changeset dir tracked.

## Decisions

- **Versions pinned:** Gradle **8.10.2**, Spring Boot **3.3.5**, dependency-management **1.1.6**. All are
  Java-21-clean and stable. No standalone Gradle was installed on the machine — the wrapper was generated by
  downloading the Gradle 8.10.2 distribution once into scratchpad; the committed wrapper is now the only
  Gradle anyone needs.
- **Compose file lives in `backend/`** (not repo root), so DB + app share one working directory. The
  step allowed either location provided the README stays consistent — see README fix below.
- **README run commands corrected** (the 0.1 handoff flagged them as forward-references to reconcile):
  changed `docker-compose` → `docker compose` (only the v2 plugin, reporting `v5.1.1`, is installed; the
  hyphenated v1 binary is **absent**) and moved `cd backend` **before** the compose call since the compose
  file is there. Added a health-check verification line.
- **`show-details: never`** on the health endpoint — the acceptance criterion only needs `{"status":"UP"}`,
  and hiding DB/disk details keeps the public health probe from leaking infra info. Revisit if a later step
  needs readiness/liveness detail.

## Acceptance criteria — all met (verified locally)

- `docker compose up -d` starts `postgres:16` (container `baseerah-postgres`), db `baseerah` on 5432,
  reaching `healthy`. ✅
- `./gradlew build` succeeds using the committed wrapper (fresh wrapper download + compile + bootJar). ✅
- `./gradlew bootRun` boots against the docker Postgres with no errors. ✅
- `GET http://localhost:8080/actuator/health` → HTTP 200 `{"status":"UP"}`. ✅
- Liquibase initialized on boot: `databasechangelog` + `databasechangeloglock` created, **0 changesets**
  applied (empty `changes/` dir loads clean). ✅
- `ddl-auto: validate` — the only tables in the DB are Liquibase's two; Hibernate created/altered nothing. ✅
