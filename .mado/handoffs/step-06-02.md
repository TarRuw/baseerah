## Key decisions

- **Cash-flow points without touching 6.1 code.** `UnderwritingReport` carries only scalar metrics and 6.2's
  file list forbids editing 6.1. So `BankService` obtains the chart series with a **second**
  `ForecastEngine.project(clientRef, 365)` call (through the *interface*, keeping the sidecar seam) and
  down-samples to one month-end point per calendar month, capped at 12. The extra projection is trivially
  cheap (whole report is 21 ms). If 6.1 is ever reopened, folding the points into `UnderwritingReport` would
  remove the double projection.
- **Report on a synthetic (unlinked) applicant â†’ 409 CONFLICT**, not 500. `BankService.report` loads the row
  first (unknown id â†’ `NotFoundException` â†’ 404) and rejects a null `clientRef` with `ConflictException`
  before scoring, so the un-underwritable synthetic fillers fail cleanly.
- **Portfolio metrics â€” all computed from seeded telemetry, no hardcoded numbers (Global Rules).** Documented
  in the `BankService` class Javadoc. The **active book** = underwritten applicants (`verdict != null`) a
  banker hasn't declined; a `DECLINE` decision drops a facility from the portfolio.
  - *Active facilities* = active-book size (**5** at seed). *Avg stamina* = its mean (**96**).
  - **NPL rate + baseline delta (the judgment curve â€” this is the documented baseline source).** A frozen
    mock has no time series, so the "âˆ’% vs baseline" is a **screened-vs-unscreened** contrast, not
    period-over-period: `nplRate` = mean 12-mo default probability of the book **after** applying the risk
    policy's auto-decline filter (`stamina >= staminaFloor AND forecastDti < autoDeclineThreshold`) â€” the
    disciplined book the bank actually carries (**9.5%**); the *baseline* is the same mean over the **whole**
    active book (approve-everyone) (**â‰ˆ11.8%**); `nplBaselineDelta = nplRate âˆ’ baseline` = **âˆ’2.3 pp**, i.e.
    pre-emptive screening (dropping the BAD/DTI-200 student) cuts modelled NPL by 2.3 pp (DESIGN Â§1 Vision-2030
    "cut bank NPLs"). Delta is always â‰¤ 0 because policy exclusions correlate with above-average PD.
  - *At-risk accounts* = active-book rows with status `AT_RISK` (**1** â€” the student).
  - *Monitoring rows*: `health` = stamina; `status` aligned to the Â§5.5 verdict (OKâ†’HEALTHY, WARNâ†’WATCH,
    BADâ†’AT_RISK); `trend` compares the row's health to the portfolio mean with a **Â±5-point deadband** (above
