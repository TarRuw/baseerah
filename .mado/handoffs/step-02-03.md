## What Was Done

Built the consumer Home/Radar screen (DESIGN §7.1) end-to-end against the live API, driven entirely by
`GET /api/v1/clients/{id}/stress-score` and `GET /api/v1/clients/{id}/accounts` — no hardcoded values.

**Data layer (dio + envelope unwrap → models)**
- `core/api_envelope.dart` — shared `unwrapEnvelope(Response)` that validates `{"status":"OK","data":...}`
  and throws `ApiEnvelopeException` on an error/mismatched envelope (one parse path for every feature).
- `features/home/data/stress_score.dart` — `StressScore` model + `StressZone` enum (`fromApi`). `color`
  is parsed from the **server-resolved `color` hex** (via `BaseerahTokens.hex`), not re-derived from the
  score, so the gauge readout can never disagree with the served zone.
- `features/home/data/account.dart`, `client.dart` — models matching `AccountDto` / `ClientDto` exactly
  (raw `latestBalance` + currency; formatting is the UI's job).
- `stress_score_repository.dart`, `accounts_repository.dart`, `client_repository.dart`.

**State (Riverpod)**
- `features/home/state/home_providers.dart` — `currentClientProvider` (resolves the **first seeded client**
  from `/clients` until a persona picker exists), `stressScoreProvider` and `accountsProvider` chained off
  it, plus `deficitSignalProvider` (see Decisions). Loading/error surface as `AsyncValue`.

**Widgets**
- `stress_gauge.dart` — `CustomPainter` 270° arc (90° bottom gap) with the three §5.1/§8 zones
  (CRITICAL 0–40 red · WARNING 40–70 orange · OPTIMAL 70–100 green) and a marker + count-up number that
  ease to the live score over **1000 ms cubic ease-out** (`BaseerahTokens.scoreAnimation` /
  `scoreAnimationCurve`). Animates from the previous score on updates.
- `deficit_warning_card.dart` — red-warning-gradient card with the **`bsr-pulse`** 2.4 s glow

- `stat_card.dart` — income-consistency / spending-velocity sub-score cards (value + proportional bar).
- `linked_accounts_list.dart` — accounts with display colour + `Fmt.money` (SAR/ر.س).
- `forecast_chart_placeholder.dart` — titled empty/skeleton slot for the Phase 3 forecast.
- `core/format.dart` — `Fmt.fmt/money` (DESIGN §8), locale-aware (Arabic-Indic digits under `ar`).

**Shell / routing**
- `shell/bottom_nav.dart` — `ConsumerBottomNav` (Home/Simulate/Rescue/Goals), extracted from the inline

## Decisions Made

- **Deficit card hidden until Phase 3 (user decision).** The real trigger is the forecast `deficit_date`
  (Phase 3 Step 3.2). Rather than fake it with an interim heuristic, `deficitSignalProvider` returns `null`,
  so the card stays hidden (Task 4/7 explicitly allows a graceful hidden state). The card + pulse are fully
  built; wiring is a drop-in — return a `DeficitSignal` from that provider once the forecast lands.
- **"Current client" = first seeded persona.** There's no persona picker in scope; `currentClientProvider`
  fetches `/clients` and takes the first. All Home data is scoped to it, keeping everything API-driven.
- **Per-section degradation.** A failed score call shows a retry in the gauge slot without taking down the
  accounts list (and vice-versa); pull-to-refresh invalidates `currentClientProvider` to refetch all.
- **Colour source of truth = API.** Gauge zone tokens are the same §8 hexes the endpoint returns; the score
  readout uses the served `color` hex directly.
