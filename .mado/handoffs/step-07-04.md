## Key decisions

- **Consumer persona selection is launch-time, not an in-app switcher** (per the user's explicit
  direction: *"each one should be with a different account so no switcher"*). Added a
  `BASEERAH_CLIENT` `String.fromEnvironment` (mirrors the existing `BASEERAH_API_BASE_URL` pattern) to
  `currentClientProvider`: unset → first seeded client (unchanged default); set → matches by
  `externalId` (case-insensitive); **unknown id fails loud** with the valid-id list. Chosen over a
  silent fallback because silently demoing the wrong persona in front of an audience is worse than a
  clear error.
- **⚠️ Scope note — one frontend code change beyond the docs-only file list.** Step 7.4's declared
  files were `docs/DEMO.md` + `README.md` only. The dry-run (Task 4) surfaced that the persona-driven
  consumer walkthrough was **not followable**: `currentClientProvider` hard-coded `clients.first` and
  no consumer persona picker existed (every screen — Home/Simulate/Rescue/Goals — followed it), so
  freelancer-Rescue / student-Goals could never be shown by clicking. I raised this to the user, who
  chose the one-account-per-persona model; the minimal `BASEERAH_CLIENT` selector is what makes the
  README/DEMO genuinely followable (the step's own acceptance criterion). No behavioural change when
  the flag is unset; no existing tests reference the provider (19/19 still green).
- **Two DESIGN §11 claims were aspirational and are documented honestly, not repeated as fact**
  (all "computed from seeded transactions", Global Rule):
  - `client_002_tech_bro` computes **OPTIMAL (~92)**, *not* a warning zone. Its real headline is the
    **large-volume performance** check (analytics returned in single-digit ms, far under the 2.5 s NFR).
    DEMO.md frames it as the perf persona with an explicit note.
  - `client_003_freelancer`'s Rescue is `hasDeficit: true` with a real ~SAR 2,779 shortfall + two
    options and a real 83→91 recovery, but `alertRaised: false` / `deficitInDays: 2603` — a
    long-horizon projection, **not** a pulsing 12–15-day alert. DEMO.md describes the rescue mechanics
    as the wow moment and calls out that the banner does not pulse on the seeded data.
