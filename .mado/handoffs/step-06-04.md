## Decisions Made

- **Persistence timing: toggles persist on change, sliders on release.** A `Switch` is a single discrete
  action → PUT immediately. A `Slider` fires many `onChanged` events per drag → keep a local draft during
  the drag (smooth UI) and PUT once on `onChangeEnd`. On save failure the control reverts to the last saved
  value (the draft is cleared and the provider's policy is the source of truth) + a retry snackbar — no
  silent failure. Inputs are disabled while a save is in flight so a second edit can't race.
- **`samaLastSync` is round-tripped, not dropped.** The backend `updateRiskPolicy` copies `samaLastSync`
  verbatim from the request DTO, so `RiskPolicy.toJson` echoes the loaded timestamp back on every PUT —
  editing a toggle/slider must not erase the last-sync time. (`toJson` emits it as a UTC ISO-8601 instant.)
- **Auto-decline slider is labelled by its real backend meaning (forecast-DTI %), range 0–200.** The
  prototype called it "Auto-decline default prob." but the authoritative `RiskPolicyDto`/`UnderwritingService`
  define `autoDeclineThreshold` as a **forecast-DTI percentage** validated 0–200 (`DTI_MAX_PERCENT`). Per the
  Global Rule against copying prototype stand-ins, the label reflects the backend semantics and the slider
  bound matches the server's `@Max(200)` so the client never submits an out-of-range value. Stamina floor is
  0–100 to match `@Max(100)`.
- **Trend arrows mirror under RTL.** `Icons.trending_up/down/flat` are direction-bearing glyphs (they point
  rightward); they're wrapped in a `Transform.flip(flipX)` when `Directionality` is RTL so "improving" reads
  the forward direction in Arabic. Up=green, down=red, flat=muted.
- **Health-score + status colours reuse the shared zone palette** (≥70 green / ≥50 orange / else red for
  health; Healthy green / Watch orange / At-risk red for status) — same thresholds as the report's stamina
  box, so both shells stay consistent (DESIGN §8). All colours come from `BaseerahTokens`.
- **All KPI/table/monitoring figures are rendered as-is from `GET /bank/portfolio`** — the client computes
  nothing (NPL rate, baseline delta, trend, status all arrive pre-derived). The verified wire contract
  matches `BankControllerTest` (field names, enum-name strings for `trend`/`status`, ISO-instant
  `samaLastSync`).
