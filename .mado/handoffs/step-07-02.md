## What Was Done

A real GenAI provider behind the existing `GenAiClient` seam (DESIGN §2/§5/§9), selectable by
`GENAI_PROVIDER=remote`, that **streams** for time-to-first-token < 1.0 s and **falls back cleanly to the mock**
when no key is present — so the demo always runs offline.

- **`RemoteGenAi`** (new) — implements `GenAiClient` and calls the provider's Messages API
  (`POST {base-url}/v1/messages`) using the JDK's built-in `java.net.http.HttpClient` + the Jackson mapper
  already on the classpath (zero new dependencies). Model `claude-opus-4-8`; headers `x-api-key`,
  `anthropic-version`, `content-type`. `chat()` streams under the hood and buffers the deltas into a full
  `ChatReply` (unchanged non-streaming contract). Grounding is built from the `ChatContext` summary only
  (persona, score, zone, monthly cash flow) — **no raw account ids** reach the provider, extending Step 7.1's
  no-leak guarantee by construction. `parseInvoice()` does a real vision call with a defensive manual-entry
  fallback so remote mode is end-to-end functional.
- **`StreamingGenAiClient`** (new) — a sibling capability interface (`streamChat(ctx, msg, onChunk)`) that only
  `RemoteGenAi` implements. This keeps the **`GenAiClient` signature frozen** (acceptance criterion) while still
  exposing token streaming.
- **`GenAiConfig`** (rewritten) — replaced the two `@ConditionalOnProperty` beans + `RemoteGenAiPlaceholder`
  with one factory `@Bean`: `provider=remote` **with a non-blank key** → `RemoteGenAi`; `remote` **without** a
  key → logs a clear warning and returns `MockGenAi` (keyless fallback); anything else → `MockGenAi`. Still
  exactly one `GenAiClient` bean, so callers inject the interface and never see a concrete type.
- **`GenAiProperties`** (new) — `@ConfigurationProperties(prefix="baseerah.genai")` binding `provider` +
  `remote.{model,base-url,api-key,max-tokens,version}`. **No secret/endpoint hardcoded** — all env-driven.
- **`application.yml`** — added `baseerah.genai.remote.*` (`GENAI_MODEL`/`GENAI_BASE_URL`/`GENAI_API_KEY`/
  `GENAI_MAX_TOKENS`/`GENAI_VERSION`), api-key defaulting to blank so mock is the offline default.
- **`ChatService.streamChat`** — `instanceof StreamingGenAiClient` → stream deltas; else emit the buffered mock
  reply as one chunk (streaming endpoint works under every provider). Reuses `buildContext` (404 contract).
- **`ChatController`** — new `POST /api/v1/clients/{id}/chat/stream` returning an `SseEmitter` (one `data:`
  event per delta), pumped off the request thread by a daemon executor. The buffered `POST .../chat` endpoint is
  untouched.
- **`GenAiProviderSelectionTest`** (new, 8 tests) — selection + keyless fallback (chat still returns a valid
  reply), interface-signature-unchanged via reflection, and streaming verified against a **local JDK
  `HttpServer` stub** (no live network): asserts per-delta chunks (`containsExactly("Hello"," world")` — proves
  no buffering) and that the outbound body carries `"stream":true`.

## Key decisions

- **Raw JDK `HttpClient` over the official Anthropic Java SDK.** The claude-api skill prefers the official SDK,
  but I deviated deliberately: the step's *Assumes* says "a WebClient/HTTP client are available", the project is
  an offline-first demo where remote is off-by-default (minimal deps), and raw `java.net.http.HttpClient` + the
  present Jackson gives correct, **verifiable** SSE streaming with zero new dependencies and no download risk.
  The wire format follows the documented Messages/SSE shape verbatim (not guessed). If a later step wants the
  SDK's retry/back-off/typed errors, swapping `RemoteGenAi`'s internals is isolated behind the interface.
- **Streaming exposed via a sibling interface, not by changing `GenAiClient`.** Acceptance requires the
  `GenAiClient` signature stay identical across providers; `StreamingGenAiClient` is the standard
  capability-detection pattern, so the mock needs no streaming code and the criterion holds.
- **Fallback is keyless, not startup-failing.** `provider=remote` with a blank/absent key warns and serves the
  mock rather than throwing — DESIGN §9 reliability ("demo works offline / without keys").
- **Model `claude-opus-4-8`, no `thinking` param.** On Opus 4.8, omitting `thinking` runs without thinking —
  the right choice for low TTFT. (No specific model was named in DESIGN §2/§5, so the provider default is used.)
