## What Was Done

Built the consumer **Rescue** screen (DESIGN §7.3, FR-06/07) as a small state machine wired to the Step 4.2
endpoints, RTL-ready in Arabic, using only shared theme tokens. Phase 4 is now complete.

- **Data layer** (`features/rescue/data/`):
  - `rescue_models.dart` — `RescueOptionType {murabaha, liquidate}` (with `fromWire`/`wire` for the
    `MURABAHA|LIQUIDATE` wire form), `RescueOption {type,label,amount,term?,detail}`, `RescueState
    {hasDeficit, deficitInDays?, alertRaised, shortfall?, options}`, `RescueOutcome
    {scoreBefore,scoreAfter,message}` — each with a `fromJson` mirroring the exact wire DTOs.
  - `rescue_repository.dart` — `fetch()` (`GET …/rescue`) and `confirm()` (`POST …/rescue/confirm` with
    `{option}`), both unwrapping the shared envelope. Widgets never touch dio (Global Rule).
- **State** (`features/rescue/state/rescue_providers.dart`) — `RescueController extends
  StateNotifier<RescueScreenState>` over a `RescuePhase {loading, error, noDeficit, open, confirming,
  complete}` enum. Named constructors make illegal states unrepresentable (no "confirming" without a
  selection). `autoDispose` + watching the client-id provider gives re-fetch-on-entry and the null→id
  rebuild for free (same pattern as Simulate's `LoanController`). `confirm()` returns a bool so a failed
  POST reverts to *open* and the screen shows a retry snackbar rather than stranding a spinner.
- **Widgets** (`features/rescue/widgets/`):
  - `shortfall_banner.dart` — dark-red `red-warning` gradient banner (−`money(shortfall)` + `in N days`);
    pulses with `bsr-pulse` **only when `alertRaised`** (honest urgency — see decisions).
  - `bridge_card.dart` — selectable card, Murabaha green `#1D9E63` / Liquidate gold `#C4A24C` icon+accent,
    shows label/detail/`money(amount)`/term; selection = accent border + tint + check.
  - `recovery_gauge.dart` — realizes the chosen recovery visual by **reusing Home's `StressGauge`**: mounts
    at `scoreBefore`, then swaps to `scoreAfter` so the gauge's own 1000 ms cubic `didUpdateWidget` tween
    plays the marker+count-up sweep. Zone colour/label derived client-side from the §5.1 thresholds.
  - `rescue_screen.dart` — header + a `switch` over the phase: spinner / error+retry / calm no-deficit /
    open (banner + "choose a bridge" + cards + gated confirm) / complete (success + recovery gauge +
    message + run-again).
- **Wiring** — `router/app_router.dart`: the placeholder Rescue branch replaced with a real
  `GoRoute → const RescueScreen()`. Bottom-nav Rescue tab already existed (Step 0.3).
- **l10n** — 12 new keys added to `app_en.arb` + `app_ar.arb` (rescue title/subtitle, shortfall label,
  `in {days} days`, choose-bridge, confirm, confirm-error, no-deficit title/body, success title, recovery
  label, run-again). **Hand-regenerated** `app_localizations.dart` / `_en.dart` / `_ar.dart` (see below).

## Decisions Made

- **Recovery visualization = animated arc-gauge sweep (user-chosen).** DESIGN §7.3 leaves this open
  ("gauge/number transition"); I presented three options and the user picked reusing `StressGauge` to sweep
  `scoreBefore → scoreAfter`. Implemented by toggling the score prop after the initial settle so the
  existing tween does the work — no new animation code, maximum visual consistency with Home.
- **Banner renders on `hasDeficit`, pulses on `alertRaised`.** Per the Step 4.1/4.2 handoffs the freelancer
  has a real deficit but `alertRaised:false` on the frozen mock. So the banner appears whenever a deficit
  exists (real shortfall + `deficitInDays`), and the `bsr-pulse` attention glow is reserved for a genuine
  15-day-lead alert — the UI never fakes urgency the engine didn't assert. (Demo caveat below.)
- **Feature-folder layout over the step's flat paths.** The step table lists flat
  `features/rescue/rescue_*.dart`, but its Notes explicitly permit matching whatever Step 0.3 established.
  Home and Simulate both use `data/ · state/ · widgets/`, so Rescue follows suit for consistency.
- **Cards show server-provided `label`/`detail` verbatim.** Same posture as chat replies (server strings);
  only the surrounding chrome is localized. If the backend serves English labels, that's a backend concern,
  not a UI-translation task in this step. Term rendered via the existing `loanTermMonths` key.
- **State modeled as an explicit phase machine**, not a bag of `AsyncValue`s — the Rescue flow is a
  sequence of discrete states, so a `RescuePhase` enum + discrete constructors fit better than Simulate's
  data-holder shape. The failed-fetch exception is retained (`failure` field) for diagnostics though the
  view shows a generic message.
