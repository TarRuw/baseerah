- **UI-03 ‚Äî recovery gauge shows the recovered score (`recovery_gauge.dart`).** The gauge previously mounted

  direction: `StressScoreCalculator` normalises **both** sub-scores so **higher = healthier** (burn-rate is
  inverted into `velocityScore`; see calculator L110/L160). Added `BaseerahTokens.subScoreColor(value)` ‚Äî a

  pointed each StatCard's **bar** (value + track) at it. The icon keeps its brand accent for identity; only the
  bar now encodes health, so a top value reads green in both cards and the bars speak the gauge's language.
- **UI-07 ‚Äî persist locale across a full reload (`locale_provider.dart`, `main.dart`, `pubspec.yaml`).** The
  locale was in-memory only, so a hard reload reverted to Arabic. Added `shared_preferences: ^2.3.2`
  (resolved 2.5.5). `main()` is now `async`: it `ensureInitialized()`s, loads `SharedPreferences` **before the
  first frame**, and injects it via a `sharedPreferencesProvider` override. `LocaleNotifier` takes the prefs,
  **restores** the saved language on construction (`_restore`, default Arabic), and **writes** the code back on
  every `set`/`toggle` ‚Äî so the choice survives a cold start with no Arabic flash.

## Key decisions

- **UI-03: rest on `scoreAfter` rather than preserve the staged before‚Üíafter sweep.** DESIGN ¬ß7.3's doc-comment
  described an `83‚Üí91` sweep, but it depended on a post-mount timer that (a) QA observed resting/reading at 83
  and (b) is starvable. The step's stated preference is "show the recovered score to match the recovery
  framing," so I dropped the two-phase dwell entirely. **Trade-off:** the count-up is now `0‚Üí91` (generic Home
  gauge motion) instead of `83‚Üí91`. A true `83‚Üí91` seed would require a `from:` parameter on `StressGauge`,
  which lives under `features/home/widgets/` ‚Äî **out of this step's scope** (spec lists `features/rescue/**`),
  so it was intentionally not touched. If the staged sweep is later deemed important, add an optional `from` to
  `StressGauge` in a Home-scoped step and have `RecoveryGauge` pass `from: scoreBefore, score: scoreAfter`.
- **UI-04: colour the bar by value, keep the icon's brand accent.** Recolouring the whole card by health would
  have changed the income card's identity too; scoping the health-band to the **bar** fixes exactly what QA
  flagged (the caution-coloured top value) while both cards keep their distinct teal/gold icons. Thresholds
  live in the theme (Global Rule) and deliberately reuse the gauge's 40/70 boundaries for one visual language.
- **UI-07: `sharedPreferencesProvider` default throws.** An un-overridden default fails loudly rather than
  silently no-op'ing persistence. Cost: any test that pumps the **real app** must supply the override ‚Äî done
  via a new `test/support/test_prefs.dart` `prefsOverride()` helper, wired into `widget_test.dart` and
  `responsive_frame_test.dart`. Screen-level tests were unaffected (they fake their data providers, so
  `apiClientProvider` ‚Äî the only other `localeProvider` reader ‚Äî is never constructed).
