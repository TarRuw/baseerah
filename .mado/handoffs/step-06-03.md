- **i18n** — 28 new bank strings added to all four l10n sources (abstract `AppLocalizations`, `_en`/`_ar`
  impls, `app_en.arb`/`app_ar.arb`), copy lifted verbatim from the prototype's ar/en tables for fidelity.
- **Test** — `test/features/bank/applications_screen_test.dart`: queue render + badges, empty→generating→

## Decisions Made

- **Feature lives under `lib/features/bank/`, not `lib/bank/`.** The step's file table suggested `lib/bank/…`,
  but the step notes explicitly say "follow whatever pattern the consumer shell established", and every
  existing feature (home/simulate/rescue/goals) sits under `lib/features/<name>/`. I followed the codebase
  convention. The bank *shell* itself remains in `router/app_router.dart` (where it already was), so no
  separate `bank_shell.dart` was created.
- **Stamina band colour is derived client-side.** The report DTO ships `staminaScore` (0–100) but no band
  label, so `_bandColor` maps it to OPTIMAL green (≥70) / WARNING orange (40–69) / CRITICAL red (<40) — the
  DESIGN §5.1 zone thresholds. This is presentational only; no report *values* are computed on the client
  (chart, KPIs, verdict all render exactly what the endpoint returns).
- **Risk badge maps from `verdict`, with an "Unscored" state for null.** A freshly-seeded applicant has a
  null verdict until a report is generated (per `ApplicantDto`), so the badge shows a neutral "Unscored"
  chip rather than guessing. OK→Low/green, WARN→Medium/orange, BAD→High/red (theme tokens only).
- **Approve is disabled for a BAD verdict** (mirrors the prototype's `approveDisabled`), and the decision
  call reverts + shows a retry snackbar on failure so a dropped backend never strands the user.
- **Decision invalidates `bankApplicantsProvider`** (with `skipLoadingOnReload: true` so the list doesn't
  flash) to reflect the persisted state. Note the row *badge* is verdict-derived, so it won't visually change
  on a decision — the persisted outcome surfaces in the detail confirmation and the retained selected
  applicant.
