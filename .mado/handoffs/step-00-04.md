# Handoff — Step 0.4: Shared API envelope, error handling & health endpoint

**Status:** completed
**Date:** 2026-07-02

## What was done

Backend-only. Established the API contract every later controller inherits (DESIGN.md §6):

- `common/ApiResponse<T>` — success envelope `{status:"OK", data}`, static `ok(data)` factory (Java record).
- `common/ApiError` — `{code, message}` record.
- `common/ApiErrorResponse` — error envelope `{status:"ERROR", error}`, static `of(code, message)` factory.
- `common/NotFoundException` — domain 404 (extends `RuntimeException`).
- `common/GlobalExceptionHandler` — `@RestControllerAdvice`, handlers ordered specific→generic:
  - `MethodArgumentNotValidException` → 400 `VALIDATION_ERROR` (aggregates field errors `field: msg`, `; `-joined)
  - `ConstraintViolationException` → 400 `VALIDATION_ERROR` (for `@RequestParam`/`@PathVariable`)
  - `NotFoundException` → 404 `NOT_FOUND`
  - `Exception` (catch-all) → 500 `INTERNAL_ERROR`, generic message — no stack trace leaked.
- `common/WebCorsConfig` — `WebMvcConfigurer`, maps `/api/**`, methods `GET,POST,PUT,DELETE,OPTIONS`,
  headers incl. `Accept-Language`/`Content-Type`. Origins from `baseerah.cors.allowed-origin-patterns`
  (default `http://localhost:*,http://127.0.0.1:*`).
- `health/HealthController` — `GET /api/v1/health` → `ApiResponse.ok({service:baseerah, status:UP})`.

Tests: `HealthControllerTest` (`@WebMvcTest` — health envelope + CORS preflight incl. Accept-Language);
`GlobalExceptionHandlerTest` (standalone MockMvc + throwaway controller — 400/404/500 paths).

## Decisions

- **Records for the envelopes** (not Lombok/POJOs) — no Lombok on the classpath; records give correct
  Jackson serialization out of the box and match the "DTOs at the edge" convention.
- **`allowedOriginPatterns` not `allowedOrigins`** — required so wildcard ports (`localhost:*`) work; Flutter
  web dev-server ports vary. `allowCredentials(true)` is compatible with patterns (not with `*` origins).
- **CORS origins are config-driven** (`@Value` with default) so emulator/device hosts can be added via
  `application.yml`/env without a rebuild — no property added to `application.yml` yet (default suffices).
- **500 message is a fixed generic string** ("An unexpected error occurred.") — deliberately does not echo
  `ex.getMessage()`, satisfying the "no stack trace / no internal detail leak" criterion.
- **Handler test uses standalone MockMvc**, not `@SpringBootTest` — avoids needing a live Postgres (JPA is on
  the classpath); isolates the advice. Health test uses `@WebMvcTest` (also DB-free) so `WebCorsConfig` loads.
- **Left `ConstraintViolationException` untested** — the acceptance criterion for validation is met by the
  `MethodArgumentNotValid` path; the constraint-violation handler needs `MethodValidationPostProcessor`
  wiring that standalone MockMvc doesn't provide. The handler is in place for real `@Validated` controllers later.

## Acceptance criteria — all met (verified by tests, `./gradlew build` green)

- `GET /api/v1/health` → 200 `{"status":"OK","data":{service,status}}`. ✅
- Validation failure → 400 `{"status":"ERROR","error":{"code":"VALIDATION_ERROR",...}}`. ✅
- `NotFoundException` → 404 `NOT_FOUND`. ✅
- Unhandled exception → 500 `INTERNAL_ERROR`, no stack trace in body. ✅
- CORS preflight from `http://localhost:3000` returns `Access-Control-Allow-Origin` and allows
  `Accept-Language`. ✅
- `./gradlew build` passes — 5 tests, 0 failures, 0 skipped. ✅
